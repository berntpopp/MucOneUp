# Bugfix Plan: Issues #79, #80, #81

## Issue #81: Amplicon expected_product_range too narrow (simplest, fix first)

**Problem:** Config default `[1500, 6000]` rejects VNTRs with >99 repeats.
**Fix:** Widen default to `[1500, 15000]` in config.json. PacBio HiFi reads 15kb+, ONT reads even longer.
**Test:** Existing amplicon tests + verify config value.

## Issue #80: Relative config paths not resolved from external directories

**Problem:** `load_config_raw()` returns paths as-is from JSON. When running from a different cwd, relative paths like `reference/model.reseq` fail.
**Fix:** Add `resolve_config_paths(config, config_dir)` call in `load_config_raw()` that makes known path fields absolute relative to the config file's parent directory. Only resolve if path is relative and doesn't already exist from cwd.
**Path keys to resolve:** `model_file` (in `pacbio_params`), `reseq_model`, `human_reference`, `sample_bam_hg38`, `sample_bam_hg19`, `sample_target_bed` (in `read_simulation`).
**Test:** Unit test that loads config from a different directory and verifies paths are resolved.

## Issue #79: ONT amplicon uses PacBio model + SAM detection gap

**Problem A:** ONT pipeline reads `pacbio_params["model_file"]` which points to a PacBio model.
**Fix A:** The CLI `--model-file` already overrides `pacbio_params.model_file`. Add validation in `ont_amplicon_pipeline.py` that warns if using a non-ONT model (check filename for "ONT" pattern). No config schema change needed — users pass `--model-file` for ONT.

**Problem B:** pbsim3 with `pass_num=1` may output `prefix_0001.sam` instead of `prefix.bam`. The wrapper only converts `prefix.sam`, missing numbered SAMs.
**Fix B:** In `pbsim3_wrapper.py` template mode output detection, add glob for `{prefix}_*.sam` and convert each to BAM before the multi-BAM check.
**Test:** Add test for numbered SAM detection and conversion.
